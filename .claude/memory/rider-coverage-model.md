The `areas` model was removed (2026-06-30). There is **no `areas` / `area_coverage_records` table and no `customers.area_id`**. A customer belongs directly to a **rider**, and rider absence reuses the existing **employee attendance/payroll** system (no separate rider-absence mechanism — `rider_absences` was created then dropped during reconciliation).

- **Relationship lives on the customer:** `customers.rider_id` (primary) + `customers.backup_rider_id` (backup, nullable), both FK `users(id)`. DB `CHECK customers_backup_ne_primary` enforces backup ≠ primary. Many customers → one rider; no join table. Set via riders-page bulk-assign, map "reassign rider", or customer form.
- **A rider is a `users` row with `role='rider'`, but its schedule/attendance/pay live on its `employees` row** (`employees.role='rider'`, `employees.user_id` → the rider's `users.id`; that user id is what `customers.rider_id` points at). Riders were backfilled into `users` from `employees`. So everywhere absence is involved, go rider-user → employee via `employees.user_id`.
- **Absence has two sources, both on the employee:**
  - **Recurring rest days** — `employees.rest_days int[]` (weekday ints `0=Sun..6=Sat`, DB default `{0}`=Sunday). Set on the employee **create/edit form** (canonical; `Employee/FormModal.vue`), also quick-edited on the riders page (`updateEmployee(id, { rest_days })`). **Payroll honors them**: `workday.ts` now takes a `restDays` param and excludes each employee's `rest_days` from paid workdays (`dailyRate = monthly ÷ that employee's workdays-in-month`), so a non-Sunday rest day no longer mis-pays. Only caller of the workday fns is `payroll.ts`.
  - **Ad-hoc absence** — an `employee_attendance` row with `status='absent'` (`upsertAttendance`, one per employee per day). The riders page "Mark absent" writes these, so a coverage-absence **also docks payroll** (the same ledger `payroll.ts` reads).
- **"Covered area" is derived, not stored:** a rider's customers grouped by their default-address barangay (`defaultBarangay`/`pickDefaultAddress` in [address.ts](../../src/helpers/address.ts); barangay from `customer_addresses` — see [[address-location-model]]). Dense barangay just splits across riders.
- **Resolution** ([riderCoverage.ts](../../src/helpers/riderCoverage.ts), replaces the old `getActiveRiderForArea`): `isRiderAbsent(schedule, status, date)` = `weekday ∈ rest_days` OR `status==='absent'`. `resolveRider(customer, scheduleByRider, attendanceByRider, date)` is **two-tier, no chaining** — primary if present → else backup if present → else `'manual'`. All **pure** over preloaded `Map`s keyed by rider **user id** (`buildScheduleByRider(riderEmployees)`, `buildAttendanceByRider(riderEmployees, attendanceForDate)` — the latter bridges `employee_attendance.employee_id` → `employees.user_id`). Build once per date, resolve client-side; never per-customer queries. Both-absent customers surface in the Delivery `NewModal` "Needs rider today" bucket. Backup whose `rest_days` collide with the primary's only **soft-warns**.
- Riders page = `/riders` (`ROUTES.RIDERS`, `src/pages/riders/index.vue`); lists rider-**employees** (`listRiderEmployees`). The old `/areas` is gone. Attendance/payroll proper still live under **Employees** (`/employees`, `employee_attendance`, `salary_records`, `payroll.ts` — salary = `monthly_salary ÷ (the employee's workdays-in-month, excluding their `rest_days`) × days_present + commission`). Marking a rider absent on the riders page writes an `employee_attendance` 'absent' row (warns-then-overwrites an existing 'present' day) — so it docks pay; rest days are skipped (not paid workdays).

**Why:** a barangay (not a named zone) is the real unit, a rider serves several, and backup-on-absence can't be a fixed secondary column — so put the rider on the customer + derive the barangay view. And since riders ARE employees with an attendance ledger + salary, rider absence must feed that one system, not a parallel table. Relates to [[tenant-auth-and-rls-model]].
